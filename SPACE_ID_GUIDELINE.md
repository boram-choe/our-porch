# 🏢 Neighborhood Beacon (여긴뭐가) - 공실 고유 ID 가이드라인

공실 데이터의 효율적인 관리와 식별을 위해 **10자리 숫자형 ID(Space ID)** 체계를 도입합니다. 이 체계는 행정 구역과 일련번호를 결합하여 중복을 방지하고 빠른 조회를 지원합니다.

## 1. ID 구조 (2-2-2-4 체계)

총 10자리의 숫자로 구성되며, 왼쪽부터 순서대로 다음과 같은 정보를 담고 있습니다.

| 섹션 | 자릿수 | 설명 | 예시 |
| :--- | :---: | :--- | :--- |
| **시 (City)** | 2 | 광역자치단체 (서울, 경기 등) | `11` (서울) |
| **구/군 (Gu)** | 2 | 기초자치단체 (서대문구, 강남구 등) | `01` (서대문구) |
| **행정동 (Dong)** | 2 | 행정복지센터 기준 행정동 | `01` (남가좌동) |
| **일련번호 (Serial)** | 4 | 해당 동네 내 등록 순서 (0001~9999) | `0005` (5번째 공실) |

**예시 ID**: `1101010005` (서울시 서대문구 남가좌동의 5번째 등록 공실)

---

## 2. 지역별 매핑 코드 (Mapping Table)

전국 행정동 데이터(`src/data/regions.json`)를 기반으로 동적으로 코드를 할당합니다. 
각 시/도, 구/군, 행정동은 고유한 2자리 코드를 가집니다.

- **시/도 (City)**: [행정표준코드](https://www.code.go.kr/)의 앞 2자리를 사용 (예: 서울 11, 부산 26, 경기 41 등)
- **구/군 (Gu)**: 해당 시/도 내에서 부여된 2자리 순번
- **행정동 (Dong)**: 해당 구/군 내에서 부여된 2자리 순번

전체 매핑 현황은 `src/data/regions.json` 파일에서 확인할 수 있습니다.

---

## 3. ID 생성 및 관리 규칙

1.  **자동 생성**: 신규 공실 등록 시 시스템이 해당 동네의 기존 공실 개수를 체크하여 자동으로 다음 번호를 부여합니다.
2.  **불변성**: 한 번 부여된 ID는 공실이 폐쇄되거나 정보가 수정되어도 변경하지 않는 것을 원칙으로 합니다.
3.  **식별 용도**: 통합 공실 관리, 전단지/지도 내 식별 번호, 민원 대응 시 고유 식별자로 활용합니다.
